Java isn't just a object-oriented programming (OOP) language, it is also a set of technologies that make software development faster and applications more robust and secure.

Java offers following benefits:

  • platform independance
  • ease of use
  • comprehensive libraries that speed up application development
  • security
  • scalability
  • extensive industry support

In tradition programming, source code is compiled to executable code. This executable code can run only on the platform it is intended to run.

A Java program, on the other hand, is compiled to bytecode. You cannot run bytecode by itself, because it isn't native code. Bytecode can only be run on a Java Virtual Machine (JVM). It's a native application that interprets bytecode.
JVM is available for many different platforms (e.g.: Windows, Unix, Linux,...), therefore code written in Java can be run on all of these without major modification.

To run a Java programm you need:

  • A compiler (javac) which converts source code to an executable format (e.g.: bytecode).
  • A JVM to run bytecode.
  • A Java Runtime Environment (JRE) which contains JVM, Java core libraries and classes
    The JRE is different for each platform.

Object-oriented programming (OOP):

OOP works by modeling applications on real-world objects. Three principles of OOP are encapsulation, inheritance and polymorphism.

Benefits of OOP:

1) Ease of maintenance
You can organize very large projects easier, because interdependency is less of a struggle. The problem with large progjects is that you a change can often have a affect on other parts of the program. OOP makes applications modular and therfore less complex.
2) Reusability
Written code can be reused for other pars with a similiar funtionality.
OOP often come with a set of ready-to-use libraries. Java contains hundreds of carefully designed and tested class libraries or application programming interfaces (APIs). That shortens development time.
3) Extendibility
Every application is unique. In OOP, extendability is achieved through inheritance. You can add functionality to existing classes. You can add methods or change existing ones. This enables you to create generic classes and add funtions for each specific application.
