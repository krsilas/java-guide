Java uses ASCII for almost all imput elements, except comments, identifiers and the contents of characters and strings.

Primitives

When writing an object-oriented application, you create an object model that resembles the real world. A payroll application, for example, needs Objects like employee, tax and company. Objects are not the only data type. There is another data type called primitive.
There are 8 primitive types in Java, each with a specific format and size.

primitive description range
byte Byte-length integer (8 Bits) -128 to 127
short Short integer (16 Bits)
int Integer (32 Bits)
long Long integer (64 Bits)
float single-presition floating point (32 Bits)
double double-presition floating point (64 Bits)
char An Unicode character See Unicode 6 specification
boolean A boolean value true or false

The first six primitives represent numbers. Each one has a different size. Some can hold larger numbers than others. We can't just use the largest primitive, because they we want to save memory space to optimize performance and disc space.
Byte and long only hold whole numbers. To store decimal numbers you need float or double. A char can contain a single Unicode character. A boolean can only store one of two values: true or false.

Variables

Variables are data placeholders. In Java every variable must have a declare type. There are two data types in Java:
- reference types: A variable of reference type provides a refernce to an object.
- primitive types: A vairable of primitive type, which holds a primitive.

In addition to the data type, a Java variable also has a name or an identifier. There are a few ground rules in choosing identifiers.
1. An identifier is an unlimited-lenth sequence of ASCII characters. An identifier must begin with a (Java) letter or an underscore.
2. An identifier must not be a Java keyword, a boolean literal or "null".
3. It must be unique within its scope.

You declare a variable by writing the type first, followed by the name plus a semicolon (";").

Constants

Constants are variables whose values cannot be changed after being assigned. You declare a constant by using the keyword final. Constants are usually written in all uppercase and words are seperated by underscores.
